---

## 📅 4周阶段性计划

### 第1周：Shell进阶巩固（3/28 - 4/3）
**主题：让脚本更健壮、更高效**

- [ ] `set -euo pipefail` 错误处理
- [ ] 理解 bash 函数返回值、局部变量
- [ ] 学会用 `xargs` 和 `find -exec` 组合
- [ ] 掌握 `awk/sed` 文本处理三剑客
- [ ] 理解 shell 中的 `||`、`&&`、`: `、`$$`、`$!`
- [ ] 每日完成1个小脚本实战

**推荐练习：**
- 批量重命名文件（正则）
- 日志分析脚本（提取IP、状态码统计）
- 备份脚本（压缩+打包+日期命名）

---

### 第2周：系统调优入门（4/4 - 4/10）
**主题：知道系统为什么会慢、为什么撑不住**

- [ ] 学会 `top/htop` 看进程，理解CPU/内存占用
- [ ] 掌握 `iostat/free/df` 做资源检查
- [ ] 学会 `ss/netstat` 看网络连接
- [ ] 理解 Swap 的原理，什么时候该加内存
- [ ] 掌握 `journalctl` 查看系统日志
- [ ] 能写一个"一键系统诊断"脚本

**推荐练习：**
- 编写系统巡检脚本（CPU+内存+磁盘+网络）
- 分析一次真实的内存告警（回顾历史cron log）

---

### 第3周：进程与信号（4/11 - 4/17）
**主题：理解Linux如何管理运行中的程序**

- [ ] 理解 `PID、PPID、父子进程`
- [ ] 学会 `kill/kill -l`，信号机制（SIGTERM/SIGKILL/SIGINT）
- [ ] 掌握 `nohup/daemonize/systemd` 守护进程
- [ ] 理解 `ps` 状态（D/S/R/Z）
- [ ] 学会用 `strace` 追踪系统调用
- [ ] 理解 `cron` 的原理和日志

**推荐练习：**
- 将一个脚本写成 systemd 服务
- 用 `strace` 分析一个简单命令的系统调用

---

### 第4周：内核与底层基础（4/18 - 4/24）
**主题：走近内核，知道边界在哪里**

- [ ] 理解 `/proc` 文件系统（cat /proc/meminfo）
- [ ] 学会看 `dmesg` 内核日志
- [ ] 理解 `sysctl` 动态调整内核参数
- [ ] 掌握 `ulimit` 资源限制
- [ ] 理解 cgroup 和容器基础（Docker原理）
- [ ] 理解"一切皆文件"的哲学

**推荐练习：**
- 用 `sysctl` 做网络参数调优（net.core.somaxconn）
- 编写一个"一键内核参数巡检"脚本
